**PEP8 (Python Enhancement Proposal 8)** – это набор рекомендаций по оформлению кода для улучшения его читаемости.
Основные правила PEP:
- •    Длина строки — не более 79 символов;
- •    Отступы — 4 пробела;
- •    Стили имён соответствуют PEP 8: Naming Conventions;
- •    Переносы строк делаются с правильными отступами;
- •    Бэкслеши \ для переносов не применяются;
- •    В коде нет неиспользуемых импортов;
- •    Импорты отсортированы в таком порядке: 
-     o    импорты стандартной библиотеки,
-     o    импорты сторонних библиотек,
-     o    импорты модулей текущего проекта.
- •    Написанные функции отделены друг от друга двумя пустыми строчками;
- •    В проекте использован одинаковый стиль кавычек: или везде одинарные, в которые вложены двойные, или везде двойные, в которые вложены одинарные;
- •    Комментарии к коду лаконичны и содержательны;
- •    Отсутствует закомментированный код.
- При именовании переменных и функций в Python принято придерживаться принципа snake_case: слова пишутся с маленькой буквы и разделены символом «_».


Git — это система контроля версий, которая предназначена для хранения информации о всех изменениях в коде/проекте.


Репозиторий – это место, где вы можете хранить свой код, файлы и историю изменения файлов.


Git предоставляет возможность работать над кодом в команде, но это касается проектов.
